四.会话控制

所谓会话控制就是:对会话进行控制

HTTP 是一种无状态的协议，它没有办法区分多次的请求是否来自于同一个客户端， 无法区分用户

而产品中又大量存在的这样的需求，所以我们需要通过 会话控制 来解决该问题

常见的会话控制技术有三种：

cookie

session

token

(一)cookie

cookie 是 HTTP 服务器发送到用户浏览器并保存在本地的一小块数据

cookie 是保存在浏览器端的一小块数据

cookie 是按照域名划分保存的

浏览器向服务器发送请求时，会自动将当前域名下,可用的 cookie 设置在请求头中，然后传递给服务器这个请求头的名字也叫 cookie ，所以将 cookie 理解为一个 HTTP 的请求头也是可以的

1.主要流程

浏览器向服务器发请求,

请求可能触发服务器设置的条件———————>向响应头设置一个键为set-cookie,值为自定义的对象

浏览器接收到含有键为set-cookie的响应头,就会在自己的请求头中设置一个键为cookie,值为set-cookie对应的值

2.express 中对cookie的操作
1)创建
2)销毁
3)读取

(二)session

session 是保存在 服务器端的一块儿数据 ，保存当前访问用户的相关信息

1)主要流程

浏览器向服务器发请求,

请求可能触发服务器设置的条件———————>创建 session 信息 ，然后将 session_id 的值通过set-cookie的响应头返回给浏览器

有了 cookie，下次发送请求时会自动携带 cookie，服务器通过 cookie 中的 session_id 的值确定用户的身份

2)express对 session 进行操作

借助express-session 创建,读取,销毁 session

借助connect-mongo将 session存入数据库中

(三)session 和 cookie 的区别

1.存在的位置

cookie：浏览器端

session：服务端

2.安全性

cookie 是以明文的方式存放在客户端的，安全性相对较低

session 存放于服务器中，所以安全性 相对 较好

3.网络传输量

cookie 设置内容过多会增大报文体积， 会影响传输效率

session 数据存储在服务器，只是通过 cookie 传递 id，所以不影响传输效率

4.存储限制

浏览器限制单个 cookie 保存的数据不能超过 4K ，且单个域名下的存储数量也有限制

session 数据存储在服务器中，所以没有这些限制

(五)token

token 是服务端生成并返回给 HTTP 客户端的一串加密字符串， token 中保存着 用户信息,主要是可以多端访问用户信息(只要携带token的字符串)

1.主要流程

1)浏览器将用户信息传递给服务器

2)服务器生成token(一串字符串)返回给浏览器.双方约定,如何携带token给服务器(一般是放在请求头中)

3)服务器根据token解析出用户信息
